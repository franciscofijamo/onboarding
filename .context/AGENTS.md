# Repository Context

This file complements the root `AGENTS.md` with details that match the current app, not the older scaffold wording.

## Working Agreement
- Install dependencies with `npm install`.
- Use `npm run dev` for the local Next.js app on port `5000`.
- Use `npm run build` before shipping changes that affect production behavior.
- Keep generated notes, prompts, and reusable task artifacts in `.context/`.

## Current Scripts
- `npm run dev` starts the app after `prisma generate`.
- `npm run build` runs `prisma generate` and builds Next.js for production.
- `npm run lint` runs the Next/ESLint configuration.
- `npm run typecheck` runs `tsc --noEmit`.
- `npm run test:unit` runs Vitest.
- `npm run test:e2e` runs Playwright against the local app.
- Database helpers live under the `db:*` scripts in `package.json`.

## Repository Map
- `components.json` stores shadcn/ui generator settings. Edit it when adding or regenerating UI primitives or changing alias/style defaults.
- `eslint.config.mjs` defines the flat ESLint config layered on top of Next.js and Prettier. Edit it when lint rules or plugin behavior change.
- `next-env.d.ts` is generated by Next.js for TypeScript support. Do not edit it manually unless a framework issue explicitly requires it.
- `next.config.ts` contains runtime and build-level Next.js options like image hosts and dev origins. Edit it when deployment or framework behavior changes.
- `package-lock.json` pins the exact npm dependency graph. It should change with dependency installs or upgrades.
- `package.json` is the source of truth for scripts, dependencies, engines, and package metadata. Edit it for dependency/script changes.
- `playwright.config.ts` defines end-to-end test behavior, browser projects, and the local web server. Edit it when e2e setup changes.
- `prisma/schema.prisma` is the source of truth for the database schema. Update it with any model or enum change, then regenerate/apply migrations.
- `prisma/generated/client/` is generated Prisma output consumed by the app. Do not hand-edit it.
- `src/app/` contains App Router pages, layouts, and API routes.
- `src/components/` contains reusable UI and feature components.
- `src/lib/` contains business logic, integrations, data access helpers, and domain code.
- `tests/e2e/` holds Playwright specs for product flows.
- `tests/unit/` holds Vitest coverage for server logic.
